# 🧭 Городской Квест-Платформа (PWA) — «Тайны Старого Города»

Интерактивное мобильное веб-приложение (Progressive Web App) для проведения городских приключенческих квестов с игровыми механиками реального времени, проверкой GPS, сканированием QR-кодов, записью аудио/видео отчетов и умным ИИ-помощником Хранителем на базе DeepSeek / Gemini.

---

## 🚀 Быстрый старт

### Требования
* Node.js (версии 18 или новее)
* npm (пакетный менеджер)

### Установка и запуск в режиме разработки
1. Установите зависимости проекта:
   ```bash
   npm install
   ```
2. Создайте файл конфигурации среды `.env` (или настройте секреты в UI AI Studio):
   ```bash
   cp .env.example .env
   ```
3. Запустите сервер разработки:
   ```bash
   npm run dev
   ```
   Приложение будет доступно по адресу: `http://localhost:3000`

---

## 🛠️ Сборка и Деплой (Production)

Проект полностью готов к деплою на любой современный full-stack или статический хостинг. 

Для сборки продакшн-версии выполните команду:
```bash
npm run build
```
Эта команда сделает два действия:
1. Соберет статические ассеты фронтенда в каталог `/dist/` с помощью Vite.
2. Скомпилирует и оптимизирует Node.js сервер в самодостаточный бандл `/dist/server.cjs` с помощью Esbuild (чтобы исключить проблемы с ES-модулями и обеспечить мгновенный холодный запуск на серверах).

### Запуск продакшн-сервера:
```bash
npm run start
```

## 📊 Структура JSON-файла квеста (`quest.json`)

Все задания квеста вынесены из кода в единый конфигурационный файл `public/data/quest.json`. Это позволяет добавлять новые этапы и изменять тексты без перекомпиляции приложения.

Пример структуры задания:
```json
{
  "id": 1,
  "type": "riddle",
  "title": "Загадка Хранителя времени",
  "description": "Первое испытание ждет вас у старой ратуши...",
  "answers": ["часы", "часики"],
  "hint": "Посмотрите на циферблат...",
  "successMessage": "Отлично! Старинные часы действительно хранят время...",
  "media": {
    "image": "images/task1_clock.jpg"
  }
}
```

### Поддерживаемые типы заданий (`type`):
1. **`riddle`** (Загадка): Текстовый ввод. Проверяется на равенство без учета регистра и пробелов.
2. **`photo`** (Фото): Камера / файл. Имитирует ИИ-сканирование Vision AI, сохраняет фото в локальном хранилище.
3. **`geolocation`** (GPS): Сравнивает координаты игрока с `coordinates` в JSON с учетом допуска `radius` (в метрах).
4. **`qrcode`** (QR-код): Камера сканирует QR и сверяет значение со списком `answers`.
5. **`audio`** (Аудиозапись): Запись голоса на встроенный микрофон, сохранение и локальное воспроизведение.
6. **`video`** (Видеозапись): Запись видео в реальном времени, воспроизведение перед отправкой.

## 🤖 Подключение и интеграция с DeepSeek

ИИ-помощник реализован по гибридной схеме повышенной надежности. Запросы обрабатываются на сервере (для сокрытия API ключей), а клиент вызывает эндпоинт `/api/chat`.

### Настройка ключей
Задайте переменные окружения в файле `.env` или в панели Secrets в AI Studio:
* `DEEPSEEK_API_KEY` — Ваш приватный API-ключ DeepSeek.
* `GEMINI_API_KEY` — Ваш API-ключ Google Gemini.

### Логика работы прокси:
1. **DeepSeek Режим**: Если в `.env` задан валидный `DEEPSEEK_API_KEY`, сервер направляет запросы к официальному API DeepSeek (`https://api.deepseek.com/chat/completions`) с моделью `deepseek-chat`.
2. **Резервный Режим (Gemini Fallback)**: Если ключ DeepSeek не предоставлен, система автоматически переключается на Google Gemini (`gemini-3.5-flash`), используя официальный SDK `@google/genai`. Это гарантирует полную работоспособность чата прямо «из коробки» в экосистеме Google AI Studio.
3. **Контекстная осведомленность**: Чат автоматически получает информацию о текущем этапе игрока (название задания, его описание, тип и игровую подсказку). ИИ проинструктирован направлять игрока, но никогда не раскрывать готовый ответ.

## 📱 PWA & Офлайн режим

* **Манифест (`manifest.json`)**: Описывает метаданные, цвета темы, иконки для главного экрана мобильного телефона, обеспечивая нативную установку (Add to Home Screen).
* **Service Worker (`sw.js`)**: Перехватывает сетевые запросы, сохраняет статические ресурсы, стили и файлы конфигурации квеста во внутренний кэш браузера. Приложение полностью работоспособно при слабом покрытии сети во время движения по городскому маршруту.
