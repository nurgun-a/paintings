# AI Quest Platform

**AI Quest Platform** — это готовая к эксплуатации (production-ready) полнофункциональная платформа для создания и проведения интерактивных AI-квестов в реальном мире. 

Платформа включает в себя мобильно-адаптированное PWA-приложение игрока с поддержкой захвата камеры, умную проверку контрольных точек с помощью ИИ (Vision AI) и текстового анализа, а также панель организатора (админку) с возможностью создания новых квестов и трансляции экстренных LIVE-событий в реальном времени.

---

## 🚀 Быстрый запуск в Docker

Проект полностью контейнеризирован и готов к запуску в любой Docker-сомместимой среде.

### 1. Переменные окружения (`.env`)
Создайте файл `.env` в корневом каталоге на основе примера:
```env
# Токен API для подключения ИИ-Ведущего и Vision AI
# Платформа использует DeepSeek как основной движок с автоматическим фоллбеком
DEEPSEEK_API_KEY="ВАШ_КЛЮЧ_DEEPSEEK"
GEMINI_API_KEY="ВАШ_КЛЮЧ_GEMINI"

# URL хостинга приложения (необходим для работы PWA и Web Push)
APP_URL="http://localhost:3000"
```

### 2. Запуск с помощью Docker Compose
Выполните следующую команду для сборки и запуска контейнеров:
```bash
docker-compose up --build -d
```
Платформа станет доступна по адресу `http://localhost:3000`.

## 📁 Структура Репозитория

Архитектура спроектирована по принципам Domain-Driven Design (DDD), SOLID и Чистой Архитектуры:

- `server.ts` — Веб-сервер Express, обслуживающий API и раздающий статику Vite.
- `/src/App.tsx` — Единый SPA-клиент (PWA Игрок + Админка Организатора).
- `/src/index.css` — Конфигурация стилей Tailwind CSS v4 с пользовательской палитрой и шрифтами.
- `/src/packages/types/` — Контракты строго типизированных моделей и схем.

## 📘 Документация OpenAPI / Swagger (HTTP Контракты)

### 1. API Игрока (Player API)

#### `GET /api/player/quests`
*Описание:* Список всех опубликованных квестов в системе.
*Ответ (200 OK):*
```json
[
  {
    "id": "spirit-of-ichchi",
    "name": "Дух Иччи (Spirit of Ichchi)",
    "description": "Мистический квест по мотивам якутской мифологии...",
    "status": "published",
    "lore": { "story": "...", "rules": "..." },
    "steps": [ ... ]
  }
]
```

#### `POST /api/player/join`
*Описание:* Регистрация игрока и начало прохождения квеста.
*Запрос:* `{"questId": "spirit-of-ichchi"}`

#### `POST /api/player/chat`
*Описание:* Отправка сообщения ИИ-Ведущему. Вызывает языковую модель `deepseek-chat` (с автоматическим фоллбеком) для генерации ответа от лица NPC с учетом уровня, инвентаря и ранга игрока. Автоматически верифицирует ответ, если текущий шаг требует текстового решения.
*Запрос:* `{"questId": "spirit-of-ichchi", "message": "Привет"}`

#### `POST /api/player/verify-photo`
*Описание:* Отправка снимка с камеры на верификацию Vision AI. Сравнивает присланный base64 снимок с эталонным описанием на сервере. Порог прохождения — 80%.
*Запрос:*
```json
{
  "questId": "spirit-of-ichchi",
  "photoBase64": "data:image/jpeg;base64,/9j/4AAQSkZJRg..."
}
```

#### `GET /api/player/events` (Server-Sent Events)
*Описание:* Подключение к каналу реального времени. Используется для мгновенного получения Live-событий от организатора.

---

### 2. API Администратора (Admin API)

#### `POST /api/admin/projects`
*Описание:* Создание и публикация нового квест-проекта с набором NPC, промптами и цепочками шагов.

#### `POST /api/admin/projects/:id/live-event`
*Описание:* Запуск экстренного интерактивного события. Рассылает push-уведомление (SSE broadcast) всем подключенным игрокам для мгновенного решения задачи и получения наград.
