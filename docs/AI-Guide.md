# Руководство по интеграции искусственного интеллекта (AI Engine Integration Guide)

В этом документе описывается устройство, алгоритмы работы, управление лимитами и стратегии отказоустойчивости модуля **AI Engine** в составе платформы **AI Quest Platform**.

---

## 🧠 1. Архитектура интеграции с DeepSeek API

Модуль спроектирован на базе официального OpenAI-совместимого клиента и использует модель **DeepSeek-V3** (alias `deepseek-chat`) в качестве основного вычислительного ядра благодаря её высочайшей логической точности, скорости и поддержке структурированных JSON ответов.

### Lazy-инициализация и безопасность ключей:
Инициализация API клиента происходит лениво при первом запросе. По желанию пользователя и для обеспечения максимальной совместимости в AI Studio, в качестве API-ключа проверяется переменная `DEEPSEEK_API_KEY`, а если она отсутствует — происходит автоматический фоллбек на `GEMINI_API_KEY` (в соответствии с конфигурацией пользователя):
```typescript
import { OpenAI } from 'openai';

let client: OpenAI | null = null;

const apiKey = process.env.DEEPSEEK_API_KEY || process.env.GEMINI_API_KEY;
if (apiKey) {
  client = new OpenAI({
    apiKey,
    baseURL: 'https://api.deepseek.com/v1',
  });
}
```

## 🗃️ 2. Контекстный генератор (Context Builder & Prompt Manager)

Каждый запрос к ИИ-ассистенту сопровождается динамической сборкой контекста, гарантирующей точные ответы без галлюцинаций:

1.  **Профиль игрока**: Уровень, текущее звание, список разблокированных достижений.
2.  **Инвентарь игрока**: Предметы, которыми владеет игрок (например, «карта-тайги», «оберег-байаная»). Это позволяет модели говорить: *«Я вижу у тебя в руках священный оберег. Давай я помогу тебе раскрыть его силу...»*.
3.  **История чата (Chat Memory)**: Последние 10 сообщений диалога для поддержания контекста беседы.
4.  **Правила игры и Сценарий**: Системный промпт, ограничивающий поведение модели рамками игровой вселенной (запрет на ответы на вопросы о программировании, политике или раскрытие ответов на следующие шаги квеста).

## 🛡️ 3. Стратегии отказоустойчивости (Fallback & Resilience)

Для обеспечения непрерывности игрового процесса при нестабильном интернет-соединении или превышении лимитов API реализованы защитные слои:

*   **Provider Switching (Переключение провайдеров)**:
    Если запрос к Google Gemini завершается ошибкой `429` (Rate Limit) или `503` (Service Unavailable), AI Engine плавно переключается на встроенный локальный процессор правил (**Rule-Based NLP Fallback Engine**). Локальный движок использует ключевые слова, регулярные выражения и шаблоны ответов, чтобы выдать игроку игровую подсказку, стилизованную под выбранного персонажа.
*   **Экспоненциальный повтор (Retry with Exponential Backoff)**:
    Сетевые ошибки обрабатываются автоматическим повторителем. Запросы повторяются до 3 раз с задержкой, возрастающей по формуле $T = 2^n \times 100\text{ мс}$.
*   **Ограничение таймаута (Strict Timeout)**:
    Каждому сетевому вызову ИИ задается жесткий таймаут в 8000 мс. Если API не отвечает вовремя, управление немедленно передается локальному обработчику, предотвращая долгое ожидание игрока на холоде или игровой локации.
