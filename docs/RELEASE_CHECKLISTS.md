# Контрольные списки подготовки к запуску (Release & Production Checklists)

В данном документе собраны обязательные чек-листы для подготовки платформы **AI Quest Platform** к запуску в промышленную эксплуатацию.

---

## 🚀 1. Контрольный список релиза (Release Checklist)
*   [ ] **Статический анализ кода**: Локально выполнены `npm run lint` и `npm run build`, ошибок и предупреждений нет.
*   [ ] **Тестирование**: Все интеграционные тесты (`npm run test` или эквивалентные скрипты) завершены со 100% успехом.
*   [ ] **Версионирование**: Обновлен номер версии в `package.json`, `Chart.yaml` и `/docs/CHANGELOG.md`.
*   [ ] **Образы контейнеров**: Собраны актуальные Docker-образы для продакшн-окружения (`gcr.io/ai-quest-platform/app:v1.0.0`).
*   [ ] **Документация**: Все инструкции (архитектура, API, администрирование, деплой) актуализированы под текущий релиз.

---

## 🏗️ 2. Контрольный список продакшна (Production Checklist)
*   [ ] **Ресурсы кластера (K8s limits)**: Заданы лимиты и запросы ресурсов (CPU, Memory) для подов во избежание Out Of Memory (OOM) сбоев.
*   [ ] **Автомасштабирование (HPA)**: Горизонтальный автоскейлер подключен к Deployment, выставлен порог триггера на 75% CPU.
*   [ ] **Балансировка сессий**: На Ingress настроена опция `sticky sessions` на куках для сохранения соединений Socket.IO.
*   [ ] **Политика миграций**: Все миграции БД выполняются автоматически перед деплоем приложения при помощи `npx prisma db push`.
*   [ ] **Связь с Redis**: Настроена отказоустойчивая шина обмена WebSocket-событиями Redis Adapter.

---

## 🗄️ 3. Контрольный список резервного копирования (Backup Checklist)
*   [ ] **Скрипт резервного копирования**: Скрипт `backup.sh` протестирован, папки и таблицы копируются корректно.
*   [ ] **Планировщик (Cron)**: В системный планировщик добавлено ежедневное задание на 2:00 ночи.
*   [ ] **Хранилище бэкапов**: Место сохранения бэкапов вынесено на отдельный изолированный диск или S3 бакет.
*   [ ] **Политика ротации**: Настроено автоматическое удаление старых резервных копий (срок хранения 7 дней) для экономии места.
*   [ ] **Тест восстановления**: Проведено успешное тестовое восстановление базы данных на пустом инстансе из файла бэкапа.

---

## 🔒 4. Контрольный список безопасности (Security Checklist)
*   [ ] **Шифрование TLS**: Активирован протокол TLS 1.3 с безопасным набором современных шифров.
*   [ ] **Заголовки безопасности**: Подключены заголовки CSP, HSTS, X-Frame-Options, X-Content-Type-Options на уровне Nginx/Traefik.
*   [ ] **Секреты и пароли**: Все пароли БД, JWT ключи и API ключи Gemini удалены из кода и перенесены в секреты Kubernetes / `.env`.
*   [ ] **Защита от XSS и SQL-инъекций**: Параметры запросов экранируются и типизируются; в текстовых вводах отключена вставка сырого HTML.
*   [ ] **Загрузка файлов (MIME)**: Настроена проверка заголовков и сигнатур загружаемых изображений. Лимит размера файла жестко ограничен 25 МБ.

---

## 📈 5. Контрольный список мониторинга (Monitoring Checklist)
*   [ ] **Scraping метрик**: Prometheus успешно опрашивает метрики приложения `/metrics` и экспортеров Postgres/Redis.
*   [ ] **Правила предупреждений (Alerts)**: В Prometheus Alerts добавлены правила для отслеживания CPU (>85%), доступной памяти (<10%) и доступности сервисов.
*   [ ] **Визуализация (Grafana)**: Настроен дашборд для отслеживания количества одновременных игроков, RPS и времени отклика ИИ.
*   [ ] **Трассировка**: Для отслеживания распределенных запросов и выявления узких мест по сети подключен трекер Grafana Tempo.
