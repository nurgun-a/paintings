# Руководство по развертыванию (Production Deployment & Scaling)

В данном документе приведено руководство по промышленному развертыванию, горизонтальному масштабированию и обеспечению безопасности инфраструктуры **AI Quest Platform**.

---

## 🏗️ 1. Требования к среде и стек технологий

Для развертывания платформы требуются следующие программные компоненты:
*   **Контейнеризация**: Docker Engine >= 20.10, Docker Compose >= 2.2.0
*   **Оркестрация**: Kubernetes >= 1.25, Helm >= 3.0
*   **СУБД**: PostgreSQL >= 15
*   **Кэш и шина событий**: Redis >= 7.0
*   **Хранилище медиафайлов**: MinIO или любое S3-совместимое облачное хранилище (например, AWS S3, Google Cloud Storage, Yandex Object Storage)

---

## ⚡ 2. Локальное и односерверное развертывание (Docker Compose)

Полное окружение (приложение, СУБД, кэш, S3, прокси и стек мониторинга) можно запустить одной командой на виртуальном сервере (VPS).

### Шаг 1: Сборка и инициализация
Выполните скрипт автоматической установки, который настроит переменные окружения, соберет Docker-образы с использованием многоэтапных файлов конфигурации, выполнит накат схем БД через Prisma и запустит контейнеры:
```bash
chmod +x deployment/scripts/*.sh
./deployment/scripts/install.sh
```

### Шаг 2: Проверка работоспособности
Убедитесь, что все компоненты функционируют исправно и отвечают на запросы:
```bash
./deployment/scripts/healthcheck.sh
```

### Доступные веб-интерфейсы по умолчанию:
*   **Nginx Proxy (Главное приложение)**: `http://localhost` (или по доменному имени)
*   **Интерфейс управления MinIO S3**: `http://localhost:9001` (Логин: `quest_minio_admin`, Пароль: `quest_minio_secret_key_2026`)
*   **Панель аналитики Grafana**: `http://localhost:3001` (Логин: `admin`, Пароль: `secure_grafana_admin_password_2026`)

## ☸️ 3. Развертывание в кластере Kubernetes

Для высокой доступности (High Availability) и отказоустойчивости рекомендуется использовать оркестратор Kubernetes.

### Шаг 1: Настройка конфигурации Helm
Отредактируйте файл `/deployment/helm/values.yaml` для указания актуальных доменов и секретных ключей.

### Шаг 2: Проверка валидности чарта
```bash
helm lint ./deployment/helm
```

### Шаг 3: Развертывание релиза
```bash
helm install quest-release ./deployment/helm --namespace ai-quest-platform --create-namespace
```

## 📈 4. Горизонтальное масштабирование и балансировка нагрузки

Платформа спроектирована с возможностью неограниченного горизонтального масштабирования (**Horizontal Scaling**):

1.  **Масштабирование бэкенда**:
    Поскольку сессии пользователей хранятся в Redis, поды бэкенда (`platform-app`) являются полностью **Stateless**. Вы можете масштабировать их от 2 до 100+ реплик на основе нагрузки CPU с помощью Horizontal Pod Autoscaler (HPA), настроенного в `/deployment/k8s/deployment.yaml`.
2.  **Масштабирование WebSockets**:
    Для синхронизации сессий реального времени между разными репликами приложения используется Redis Adapter (`@socket.io/redis-adapter`). Любое событие, отправленное в комнату на реплике А, будет мгновенно доставлено клиенту, подключенному к реплике Б, через встроенный механизм Redis Pub/Sub.
3.  **Sticky Sessions (Липкие сессии)**:
    При использовании транспорта HTTP Long Polling в Socket.IO (в случае, если WebSocket временно заблокирован корпоративным файрволом), балансировщик нагрузки (Ingress) **обязан** поддерживать липкие сессии на основе cookies для направления запросов рукопожатия к одной и той же реплике:
    ```yaml
    nginx.ingress.kubernetes.io/affinity: "cookie"
    nginx.ingress.kubernetes.io/session-cookie-name: "route"
    ```

## 🔒 5. Безопасность и TLS Hardening (Усиление защиты)

Для защиты пользовательских данных и предотвращения атак в инфраструктуре реализованы следующие защитные механизмы:

*   **Шифрование TLS 1.3**:
    В конфигурации Traefik (`/deployment/traefik/traefik.yml`) и Nginx принудительно отключены устаревшие версии TLS (1.0, 1.1, 1.2) и слабые шифры. Разрешены только современные криптографические алгоритмы (например, `TLS_AES_256_GCM_SHA384`).
*   **HSTS (HTTP Strict Transport Security)**:
    Заголовки безопасности принуждают браузеры взаимодействовать с платформой исключительно по зашифрованному HTTPS-каналу.
*   **MIME-Type Validation & Upload Protection**:
    При загрузке фотографий для проверки ИИ-зрением, загружаемые файлы проверяются на уровне MIME-сигнатур и ограничиваются размером в 25 МБ во избежание атак типа Denial of Service (DoS).
*   **Защита от Replay-атак**:
    Шлюз Socket.IO кэширует уникальные UUID транзакционных сообщений в течение скользящего 10-минутного окна, отклоняя любые дублирующиеся попытки передачи одних и тех же игровых событий.
